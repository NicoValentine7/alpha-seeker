# スコアリング方法論 詳細ドキュメント

## 1. スコアリングの全体構造

### 1.1 設計思想

本システムは「伸び代のある過小評価株」を発掘することを目的とする。地政学的リスクや構造的衰退による恒久的な低バリュエーションではなく、**市場がまだ十分に織り込んでいない成長ポテンシャル**を持つ銘柄を特定する。

この目的を達成するため、以下の4カテゴリのスコアを**セクター内パーセンタイルランク（0-100）**で算出し、重み付き合成する。

### 1.2 4カテゴリモデル

```
総合スコア = バリュエーション(25%) + 成長力(30%) + 質(20%) + 決算モメンタム(25%)
```

**なぜこの4カテゴリか：**
- **バリュエーション**: 現在の株価が内在価値に対して割安かを測定
- **成長力**: 企業の将来の拡大余地を測定（「伸び代」の直接的指標）
- **質**: 成長の持続可能性と財務健全性を測定（バリュートラップ排除にも寄与）
- **決算モメンタム**: 市場が認識し始めている改善シグナルを捕捉（最もタイムリーな指標）

**なぜこの重み配分か：**
- 成長力を最も重視（30%）: 「伸び代のある株」を発掘する目的に直結
- 決算モメンタム（25%）: EPS修正が最強の短期予測ファクターであるため（Zacks, Mill Street Research）
- バリュエーション（25%）: 安さは必要条件だが十分条件ではない
- 質（20%）: 品質は長期保有のフィルターとして機能

### 1.3 セクター内相対評価（パーセンタイルランク）

各指標は**同一セクター内**でのパーセンタイルランク（0-100）に変換される。

**なぜ絶対値ではなくセクター内相対か：**
- テクノロジーセクターのPER中央値30-40倍 vs 金融セクター10-15倍
- 絶対閾値（PER<15が割安）では特定セクターが体系的に過大/過小評価される
- Stockopedia, Seeking Alpha, Koyfin すべてがセクター内相対を採用

**外れ値処理：**
- 5-95パーセンタイルでクリッピング（極端な値の影響を抑制）
- セクター内銘柄数3未満のセクターはスキップ

## 2. バリュエーションスコア（25%）

### 2.1 指標と重み

| 指標 | 重み | 方向 | データソース |
|------|------|------|-------------|
| PER (Trailing P/E) | 20% | 低いほど良い | `info.trailingPE` |
| PBR (Price/Book) | 15% | 低いほど良い | `info.priceToBook` |
| EV/EBITDA | 20% | 低いほど良い | `info.enterpriseToEbitda` |
| PSR (Price/Sales) | 15% | 低いほど良い | `info.priceToSalesTrailing12Months` |
| **FCF利回り (FCF/EV)** | **30%** | **高いほど良い** | `info.freeCashflow / info.enterpriseValue` |

### 2.2 FCF利回りを最高ウェイトにした根拠

Pacer ETFsの30年バックテストで、FCF利回り（FCF / Enterprise Value）が全バリュエーション指標中で最も高いリターンと最も少ないマイナスリターン期間を記録。Alpha Architectの40年バックテスト（1971-2010）でも、EBITDA/EV（年17.7%）とFCF/TEV（年16.6%）がE/M（PERの逆数）を上回った。

FCF利回りがPERより優れる構造的理由：
- 会計上の利益（EPS）は減価償却方法・一時項目で操作可能だが、キャッシュフローは操作困難
- Enterprise Valueを分母に使うため負債構造の違いも考慮される
- Warren Buffettの「Owner Earnings」概念と整合

### 2.3 PEG比率について

PEG = Forward PE / EPS成長率(%)。GARP戦略の核心指標で、PEG<1が「成長に対して割安」の目安。
ただしPEG単独の限界があるため（赤字企業に使えない、成長率推定のバイアス）、「成長力」カテゴリの一指標として15%の重みで組み込んでいる。

## 3. 成長力スコア（30%）

### 3.1 指標と重み

| 指標 | 重み | 方向 | データソース |
|------|------|------|-------------|
| 売上成長率 (YoY) | 30% | 高いほど良い | 財務諸表から計算（`revenue_growth_calc`）、なければ`info.revenueGrowth` |
| 営業利益成長率 (YoY) | 25% | 高いほど良い | 財務諸表から計算 |
| EPS成長率 (YoY) | 30% | 高いほど良い | 財務諸表から計算、なければ`info.earningsGrowth` |
| **PEG比率** | **15%** | **低いほど良い** | `forward_pe / (earningsGrowth * 100)` |

### 3.2 成長率の計算方法

成長率は2つのソースから取得可能：
1. **財務諸表計算値**（優先）: `t.financials` から直近2年の値を取得して計算
2. **info APIの値**（フォールバック）: yfinanceが提供する `revenueGrowth`, `earningsGrowth`

財務諸表計算値を優先する理由は、infoの値は更新タイミングが不明確な場合があるため。

## 4. 質・健全性スコア（20%）

### 4.1 指標と重み

| 指標 | 重み | 方向 | データソース |
|------|------|------|-------------|
| ROE (Return on Equity) | 30% | 高いほど良い | `info.returnOnEquity` |
| **粗利率 (Gross Margin)** | **20%** | **高いほど良い** | `info.grossMargins` |
| D/E比率 (Debt/Equity) | 25% | 低いほど良い | `info.debtToEquity` |
| FCFマージン (FCF/Revenue) | 25% | 高いほど良い | 財務諸表から計算 |

### 4.2 粗利率を追加した根拠

Novy-Marx (2013, Journal of Financial Economics) の研究で、粗利益/総資産（Gross Profitability）がP/B比率と同等のリターン予測力を持つことが発見された。この知見はFama-Frenchの5ファクターモデルへの収益性ファクター追加を促した。

粗利率はMorningstarの「経済モート」のプロキシとしても機能する：
- 高い粗利率の持続 → コスト優位性またはブランド力の証拠
- Morningstarの5つのモートソース（コスト優位性、無形資産、ネットワーク効果、スイッチングコスト、効率的規模）のうち、コスト優位性を近似

## 5. 決算モメンタムスコア（25%）

### 5.1 指標と重み

| 指標 | 重み | 方向 | データソース |
|------|------|------|-------------|
| 平均決算サプライズ率 | 25% | 高いほど良い | `earnings_dates` の `Surprise(%)` 直近4Q平均 |
| EPS予想修正率（90日間） | 25% | 高いほど良い | `eps_trend` の current vs 90daysAgo |
| 売上成長加速度 | 20% | 高いほど良い | 直近QoQ成長率 - 前回QoQ成長率 |
| 来期EPS成長予想 | 30% | 高いほど良い | `earnings_estimate` の +1y growth |

### 5.2 なぜ決算モメンタムが重要か

**PEAD (Post-Earnings Announcement Drift):**
- 1968年にBall & Brownが文書化した最も古く持続的な市場アノマリー
- 好決算の企業の株価が発表後60日以上にわたって上昇し続ける現象
- Garfinkel et al. (2024): CNN分析で最上位・最下位デシル間で63日間で3.6%のリターン差

**EPS修正モメンタムの予測力:**
- Zacks Rank: EPS予想修正を株価に最も影響するファクターと結論
- Mill Street Research: 修正のbreadth(幅)とmagnitude(大きさ)が1-6ヶ月のリターンと相関
- 強い上方修正の銘柄は低ボラティリティも示す（リスク調整後でも優秀）

### 5.3 売上成長加速度について

`revenue_acceleration = 直近QoQ成長率 - 前回QoQ成長率`

他のプラットフォームでは明示的に使われていない独自指標。成長の「変化率」（2階微分）は「水準」（1階微分）よりも将来のサプライズを予測する可能性がある。正の加速度は成長が加速中であることを示し、市場がまだ織り込んでいない改善を捕捉できる。

## 6. バリュートラップフィルター（失格ルール）

### 6.1 フィルター条件

| 条件 | 閾値 | ペナルティ |
|------|------|-----------|
| 四半期売上の連続減少 | 3四半期連続 | 総合スコア -20点 |
| D/E比率の極端な高さ | 500%超 | 総合スコア -20点 |

### 6.2 導入根拠

- **Piotroski (2000)**: バリュー株の約57%が1-2年で市場をアンダーパフォーム
- **Seeking Alpha**: いずれかのファクターでD+以下の銘柄はNeutral以上にならない「失格ルール」
- **TIKR.com**: 売上の停滞・減少がバリュートラップの最大レッドフラグ
- **Lord Abbett (2025)**: 相対バリュエーションだけでなく、ビジネスの質・ファンダメンタルトレンドの安定性も必須

## 7. 定性的評価について

### 7.1 レポートに含まれる定性情報

スコアリングは定量的だが、レポートには以下の定性情報を含めて投資家の判断材料とする：

1. **決算情報セクション**: 四半期ごとのEPS予想vs実績、サプライズ率推移、ビート回数
2. **アナリスト評価**: コンセンサス評価、ターゲット価格、上昇余地
3. **直近アナリストアクション**: アップグレード/ダウングレードの詳細（日付・証券会社・ターゲット価格変更）
4. **直近ニュース**: タイトル・日付・発行元・URL（最大5件）
5. **スコア根拠サマリー**: 自然言語でスコアの理由を説明

### 7.2 将来の定性評価拡張候補

- ニュースセンチメント分析（FinBERT）: 学術的には有効だが、ロジスティック回帰に劣る場合もある（Haq et al. 2023）
- インサイダー取引シグナル: SEC EDGAR Form 4から取得可能、特に「クラスター買い」が強力
- Morningstarモート評価のプロキシ: 粗利率安定性・ROE持続性で近似（現在は粗利率の水準のみ）

## 8. データソース一覧

| データ | ソース | 取得方法 | 更新頻度 |
|--------|--------|----------|----------|
| バリュエーション指標 | yfinance `info` | API | リアルタイム |
| 財務諸表（年次） | yfinance `financials` | API | 四半期決算時 |
| 四半期決算 | yfinance `quarterly_financials` | API | 四半期決算時 |
| 決算サプライズ | yfinance `earnings_dates` | Webスクレイプ | 決算後 |
| EPS予想トレンド | yfinance `eps_trend` | API | 随時 |
| 来期成長予想 | yfinance `earnings_estimate`, `revenue_estimate` | API | 随時 |
| アナリストアクション | yfinance `upgrades_downgrades` | API | 随時 |
| ニュース | yfinance `news` | API | リアルタイム |
| S&P500銘柄リスト | Wikipedia | Webスクレイプ | 随時 |

### 8.1 yfinance APIの注意点

- `earnings_dates` は `lxml` パッケージが必要（HTMLパーシング）
- `eps_trend` は今期(0y)と来期(+1y)の4時点（current, 7d, 30d, 60d, 90d前）を提供
- `info` の成長率指標（`revenueGrowth`, `earningsGrowth`）は更新タイミングが不明確な場合がある
- API制限を避けるため、リクエスト間隔0.15秒（`FETCH_DELAY_SEC`）を設定
